Guided onboarding through the OpenSpec workflow using your actual Rust codebase.

**Steps**

1. **Welcome and scan** — analyze twisterDBA codebase for a small improvement
2. **Find opportunity** — suggest a small, safe change (e.g., add a keybinding, improve error message)
3. **Create change** — `/opsx-new`
4. **Write proposal** — explain why and what
5. **Create specs** — requirements with scenarios
6. **Write design** — technical approach
7. **Create tasks** — implementation checklist with cargo verification
8. **Implement** — work through tasks with `/opsx-apply`
9. **Verify** — `/opsx-verify` with cargo checks
10. **Archive** — `/opsx-archive`

Takes 15-30 minutes using real code, not toy examples.
